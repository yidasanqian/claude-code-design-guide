# 前言：为什么要读这本书

## 一个问题

2025 年，如果你问一个程序员："你用 AI 写代码吗？"

大概率得到的回答是："用，但感觉就是个高级自动补全。"

这个回答揭示了一个认知断层：大多数人把 AI 编程工具当成了一个更聪明的 IDE 插件，而不是一个真正的**工程系统**。

Claude Code 打破了这个认知。

---

## Claude Code 不是聊天机器人

当你第一次打开 Claude Code，你会看到一个朴素的终端界面。没有花哨的 GUI，没有拖拽操作，只有一个命令行提示符。

但在这个朴素的界面背后，是一套精心设计的工程系统：

- 一个能够**读写文件、执行命令、搜索代码**的工具引擎
- 一个管理**对话上下文、压缩历史、注入记忆**的上下文系统
- 一个支持**多代理协作、后台任务、远程执行**的 Agent Runtime
- 一个通过 **MCP 协议**连接外部世界的扩展系统
- 一套从**权限检查到沙箱隔离**的安全模型

这不是一个聊天机器人。这是一个 **AI 原生的工程平台**。

---

## 关于本书的源码基础

**2025 年 3 月，Anthropic 官方 Claude Code 的完整 TypeScript 源代码意外泄漏**。这份源码是 `src/` 目录下完整的、未经混淆的工程代码，包含：

- 完整的类型定义和接口
- 真实的业务逻辑与架构决策
- 开发者注释和 TODO 标注
- 内部 feature flags 和实验性功能

这让本书得以从**第一手源码**出发，而不是从 npm 编译产物中推断设计意图。书中引用的每一段代码，都来自这份真实的源码，不做猜测，不做过度解读。

> **注意**：源码的获取和研究出于学习和工程参考目的。书中对设计思想的分析和提炼，是这份源码最有价值的转化。

---

## 为什么要深入这份源码

读源码的价值不在于"抄作业"，而在于：

1. **理解设计决策背后的权衡**：为什么这样设计工具系统？权限模型为什么分五层？Context Engineering 的工程实现是什么？
2. **学习现代 Agent 工程方法**：Feature flags、流式处理、权限分层、状态管理——这些都是工业级 AI 系统的真实实践
3. **建立 Agent 系统的心智模型**：什么是 Agent Runtime？多代理如何协调？上下文如何压缩？

这些问题，只有完整源码才能给出真实答案。

---

## 这本书的结构

这本书分为九个部分，从浅到深：

**第一部分**给小白看，解释 Claude Code 是什么、能做什么。

**第二部分**追溯历史，从 Unix 哲学、REPL 演化到 AI Agent 的诞生，建立认知框架。

**第三到八部分**是核心，深入分析架构设计、工具系统、Context Engineering、Agent Runtime、扩展系统、安全模型。

**第九部分**提炼设计哲学，总结可以迁移到你自己项目中的工程原则。

---

## 写给不同读者的话

**如果你是小白**：不要被"源码分析"吓到。第一、二部分完全不需要编程基础，你只需要好奇心。

**如果你是开发者**：重点在第三到八部分。每一章都有真实的源码片段和设计分析，你可以直接对照自己的项目思考。

**如果你在设计 Agent 系统**：第五章（从聊天机器人到 Agent）、第六章（查询引擎）、第十二到十五章（Context Engineering）、第十六到十八章（Agent Runtime）是你最需要的内容。
