# 제2장: 빠른 시작

> 배우는 가장 좋은 방법은 직접 해보는 것입니다.

---

## 2.1 설치

Claude Code는 Node.js 18+ 또는 Bun 런타임이 필요합니다.

```bash
# npm으로 설치
npm install -g @anthropic-ai/claude-code

# 설치 확인
claude --version
```

첫 실행 시 API Key 설정이 필요합니다:

```bash
claude
# Anthropic API Key를 입력하라는 안내가 나타납니다
# 또는 환경 변수 설정
export ANTHROPIC_API_KEY=sk-ant-...
```

---

## 2.2 세 가지 사용 모드

### 모드 1: 대화형 REPL

가장 일반적인 모드로, Claude와 대화하듯 사용합니다:

```bash
claude
```

진입 후 다음을 할 수 있습니다:
- 질문이나 작업을 직접 입력
- `/` 접두사로 슬래시 커맨드 사용
- `Ctrl+C`로 현재 작업 중단
- `Ctrl+D` 또는 `/exit` 입력으로 종료

### 모드 2: 단일 실행

스크립트 통합이나 빠른 작업에 적합합니다:

```bash
# 작업 직접 실행
claude "이 프로젝트의 구조를 설명해줘"

# 비대화형 모드 (사용자 입력을 기다리지 않음)
claude --print "모든 TypeScript 파일을 나열해줘"
```

### 모드 3: 파이프라인 모드(Pipeline Mode)

표준 입력에서 내용을 읽습니다:

```bash
# git diff 분석
git diff | claude "이 변경사항을 설명해줘"

# 로그 분석
cat error.log | claude "오류의 원인을 찾아줘"
```

## 2.3 주요 슬래시 커맨드

대화형 모드에서 슬래시 커맨드는 Claude Code의 동작을 제어하는 단축키입니다:

| 명령어 | 용도 |
|--------|------|
| `/help` | 도움말 표시 |
| `/clear` | 대화 히스토리 삭제 |
| `/compact` | 대화 컨텍스트 압축 |
| `/cost` | 이 대화의 토큰 사용량 및 비용 표시 |
| `/config` | 설정 확인 또는 수정 |
| `/model` | 모델 전환 |
| `/commit` | git 커밋 메시지 생성 및 커밋 |
| `/exit` | 종료 |

## 2.4 첫 번째 실습: 프로젝트 구조 분석

새로운 프로젝트가 있고 빠르게 이해하고 싶다고 가정합니다:

```
> 이 프로젝트의 전체 구조를 분석해줘. 주요 모듈과 각각의 역할을 포함해서
```

Claude Code는 다음을 수행합니다:
1. `GlobTool`을 사용하여 파일 구조 스캔
2. 핵심 파일 읽기 (`package.json`, `README.md`, 진입 파일)
3. 명확한 프로젝트 개요 제공

---

## 2.5 두 번째 실습: 버그 수정

```
> npm test를 실행하고, 실패하는 테스트를 찾아서 수정해줘
```

Claude Code는 다음을 수행합니다:
1. `npm test` 실행
2. 오류 출력 분석
3. 관련 소스 파일 위치 확인
4. 코드 수정
5. 테스트를 다시 실행하여 검증

## 2.7 CLAUDE.md: Claude를 위한 지침서

프로젝트 루트 디렉토리에 `CLAUDE.md` 파일을 생성합니다. Claude Code는 시작할 때마다 이 파일을 읽어 프로젝트 컨텍스트로 사용합니다:

```markdown
# 프로젝트 설명

## 기술 스택
- Node.js 20 + TypeScript
- PostgreSQL + Prisma ORM
- Jest 테스트 프레임워크

## 코드 규칙
- ESLint + Prettier 사용
- 함수명은 camelCase 사용
- 파일명은 kebab-case 사용

## 주요 명령어
- `npm run dev` 개발 서버 시작
- `npm test` 테스트 실행
- `npm run build` 프로덕션 빌드

## 주의사항
- generated/ 디렉토리의 파일을 직접 수정하지 마세요
- 데이터베이스 마이그레이션은 스테이징 환경에서 먼저 검증해야 합니다
```

이것은 Claude Code의 가장 중요한 설정 중 하나입니다. 좋은 `CLAUDE.md`는 Claude가 프로젝트를 더 정확하게 이해하고 실수를 줄이는 데 도움을 줍니다.

## 2.8 권한 설정

Claude Code는 기본적으로 위험한 작업을 실행하기 전에 물어봅니다. 설정을 통해 조정할 수 있습니다:

```bash
# 현재 권한 설정 확인
claude /config

# 자동 bash 명령어 실행 허용 (주의해서 사용)
claude --dangerously-skip-permissions
```

권한 수준 (낮은 것에서 높은 순):
1. **기본**: 위험 작업 전 확인 요청
2. **자동 승인**: 특정 도구 자동 승인
3. **권한 건너뛰기**: 모든 작업 자동 실행 (신뢰할 수 있는 자동화 시나리오에서만 사용)

## 2.9 멀티 파일 프로젝트 모범 사례

**Claude에게 충분한 컨텍스트를 제공하세요**:
```
> 사용자 인증 흐름을 수정하고 싶어. 관련 파일은 src/auth/ 디렉토리에 있고,
  데이터베이스 스키마는 prisma/schema.prisma에 있어.
  수정을 시작하기 전에 이 파일들을 먼저 읽어줘
```

**복잡한 작업은 단계별로 실행하세요**:
```
> 1단계: 먼저 기존 인증 흐름을 분석해줘, 파일은 수정하지 마
> (분석이 정확한지 확인 후)
> 2단계: 이제 우리가 논의한 계획대로 수정해줘
```

**`/compact`로 컨텍스트를 관리하세요**:
긴 대화 후에 `/compact`를 사용하여 히스토리를 압축하고, 핵심 정보를 유지하면서 토큰 공간을 확보합니다.
