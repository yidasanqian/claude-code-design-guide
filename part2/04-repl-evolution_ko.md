# 제4장: REPL의 진화

> "읽기-평가-출력 루프는 가장 단순한 인터랙티브 프로그래밍 환경입니다."

---

## 4.1 REPL이란 무엇인가

REPL은 **Read-Eval-Print Loop**의 약자입니다.

1. **Read(읽기)**: 사용자 입력을 읽습니다
2. **Eval(평가)**: 실행/평가합니다
3. **Print(출력)**: 결과를 출력합니다
4. **Loop(반복)**: 첫 번째 단계로 돌아갑니다

이것은 가장 단순한 인터랙티브 프로그래밍 환경입니다. 코드 한 줄을 입력하면 즉시 결과를 확인할 수 있습니다.

```python
>>> 1 + 1
2
>>> "hello".upper()
'HELLO'
>>> [x**2 for x in range(5)]
[0, 1, 4, 9, 16]
```

---

## 4.2 REPL의 기원: Lisp (1958)

REPL 개념은 1958년 John McCarthy가 발명한 Lisp에서 유래합니다.

Lisp의 REPL은 기존의 "작성-컴파일-실행" 사이클을 깨뜨려 프로그래머가 **즉시 탐색**할 수 있게 했다는 점에서 혁명적이었습니다.

```lisp
; Lisp REPL, 1960년대
> (+ 1 2)
3
> (defun square (x) (* x x))
SQUARE
> (square 5)
25
```

이 "즉각적인 피드백"이라는 아이디어는 모든 현대 인터랙티브 도구의 기반입니다.

---

## 4.3 셸(Shell): 운영체제의 REPL

유닉스 셸(sh, bash, zsh)은 운영체제 수준의 REPL입니다.

```bash
$ ls -la
$ cd src
$ grep -r "TODO" .
$ git status
```

셸의 REPL 루프:
- **Read(읽기)**: 명령줄 입력을 읽습니다
- **Eval(평가)**: 명령을 파싱하고, 서브프로세스를 포크하여 실행합니다
- **Print(출력)**: 출력을 표시합니다
- **Loop(반복)**: 새로운 프롬프트를 표시합니다

셸의 위대함은 간단한 텍스트 인터페이스를 통해 사용자에게 **운영체제 기능**을 노출한다는 데 있습니다.

---

## 4.4 Node.js REPL: JavaScript의 즉석 환경

2009년, Node.js는 서버 사이드 JavaScript와 Node.js REPL을 가져왔습니다.

```javascript
$ node
> const arr = [1, 2, 3, 4, 5]
undefined
> arr.filter(x => x % 2 === 0)
[ 2, 4 ]
> arr.reduce((sum, x) => sum + x, 0)
15
```

Node.js REPL의 기능:
- 다중 줄 입력 지원
- 자동 완성
- 히스토리
- 모듈 `require` 가능

## 4.5 IPython/Jupyter: 과학 컴퓨팅의 REPL

2001년, Fernando Pérez가 IPython을 만들었고, 이것은 나중에 Jupyter Notebook으로 발전했습니다.

Jupyter는 REPL 개념을 새로운 차원으로 끌어올렸습니다.

```python
# 셀 1
import pandas as pd
df = pd.read_csv('data.csv')
df.head()
# 즉시 테이블을 표시합니다

# 셀 2
df.describe()
# 즉시 통계를 표시합니다

# 셀 3
df.plot(kind='bar')
# 즉시 차트를 표시합니다
```

Jupyter의 혁신:
- **셀 모델**: 코드가 블록 단위로 실행되며, 각각 독립적인 출력을 가집니다
- **풍부한 미디어 출력**: 텍스트뿐만 아니라 차트, 테이블, HTML
- **영속적인 상태(State)**: 셀 간에 변수가 공유됩니다
- **내러티브**: 코드와 문서가 혼합됩니다

Claude Code는 `NotebookEditTool`을 지원하여 Jupyter Notebook을 직접 편집할 수 있습니다. Notebook이 데이터 과학 워크플로의 중심이 되었기 때문입니다.

## 4.6 ChatGPT: 대화로서의 REPL

2022년, ChatGPT는 REPL 개념을 자연어 수준으로 가져왔습니다.

```
사용자: 퀵소트 작성을 도와줘
ChatGPT: [코드 제공]

사용자: 커스텀 비교 함수를 지원하도록 수정해줘
ChatGPT: [코드 수정]

사용자: 단위 테스트를 추가해줘
ChatGPT: [테스트 추가]
```

이것은 새로운 유형의 REPL입니다.
- **Read(읽기)**: 자연어 입력을 읽습니다
- **Eval(평가)**: LLM이 이해하고 응답을 생성합니다
- **Print(출력)**: 텍스트/코드를 출력합니다
- **Loop(반복)**: 대화를 계속합니다

그러나 ChatGPT의 REPL에는 근본적인 한계가 있습니다. **텍스트만 생성할 수 있고, 액션을 실행할 수 없습니다**.

---

## 4.7 Claude Code: 액션으로서의 REPL

Claude Code는 REPL 진화의 다음 단계입니다. **자연어 REPL과 도구 실행의 결합**입니다.

```
사용자: 프로젝트에서 사용하지 않는 임포트를 모두 찾아서 삭제해줘

Claude Code:
  → GlobTool: 모든 .ts 파일 찾기
  → FileReadTool: 각 파일 읽기
  → 사용하지 않는 임포트 분석
  → FileEditTool: 사용하지 않는 임포트 삭제
  → BashTool: tsc 실행하여 컴파일 통과 검증
  → 완료, 23개 파일 수정
```

이 REPL의 특성:
- **Read(읽기)**: 자연어 의도를 읽습니다
- **Eval(평가)**: 도구 호출 체인을 계획하고 실행합니다
- **Print(출력)**: 실행 과정과 결과를 표시합니다
- **Loop(반복)**: 다음 지시를 기다립니다

---

## 4.8 REPL 진화의 핵심 차원

| 차원 | Lisp REPL | 셸(Shell) | Jupyter | ChatGPT | Claude Code |
|------|-----------|-----------|---------|---------|-------------|
| 입력 유형 | 코드 | 명령어 | 코드 | 자연어 | 자연어 |
| 실행 능력 | 연산 | 시스템 운영 | 연산+시각화 | 없음 | 시스템 운영+연산+네트워크 |
| 상태(State) 지속성 | 세션 | 세션 | 노트북 | 대화 | 대화+파일 시스템 |
| 출력 유형 | 값 | 텍스트 | 풍부한 미디어 | 텍스트 | 텍스트+액션 결과 |
| 조합 가능성 | 낮음 | 높음(파이프) | 중간 | 낮음 | 높음(도구 체인) |

## 4.9 Claude Code의 REPL 설계

Claude Code의 REPL 구현은 `src/entrypoints/cli.tsx`에 있으며, 핵심 루프는 대략 다음과 같습니다.

```
1. 프롬프트를 표시하고 사용자 입력을 기다립니다
2. 입력을 파싱합니다(슬래시 커맨드(Slash Command) 또는 자연어)
3. 슬래시 커맨드(Slash Command)면 직접 실행합니다
4. 자연어면 쿼리 엔진(QueryEngine)에 제출합니다
5. 쿼리 엔진(QueryEngine)이 Claude API를 호출하고 스트리밍(Streaming) 응답을 받습니다
6. 응답에 도구 호출이 포함되면 도구를 실행합니다
7. 도구 결과가 대화에 추가되고, 생성이 계속됩니다
8. 최종 결과를 표시합니다
9. 1단계로 돌아갑니다
```

이 루프에는 몇 가지 핵심 설계 특징이 있습니다.

**스트리밍(Streaming) 표시**: Claude의 응답은 스트리밍되므로, 사용자는 완전한 응답을 기다리지 않고 내용을 볼 수 있습니다.

**도구 호출 투명성**: 모든 도구 호출이 사용자에게 표시되므로, Claude가 무엇을 하고 있는지 확인할 수 있습니다.

**중단 가능**: 사용자는 언제든지 `Ctrl+C`를 눌러 현재 작업을 중단할 수 있습니다.

**히스토리**: 대화 히스토리가 로컬에 저장되어 이전 세션을 복원할 수 있습니다.

---

## 4.10 요약

REPL의 진화는 **인터랙션 장벽을 낮춘** 역사입니다.

- Lisp REPL: 프로그래머가 코드를 즉시 탐색할 수 있게 했습니다
- 셸(Shell): 사용자가 운영체제를 즉시 조작할 수 있게 했습니다
- Jupyter: 데이터 과학자가 데이터를 즉시 탐색할 수 있게 했습니다
- ChatGPT: 일반인이 자연어로 인터랙션할 수 있게 했습니다
- Claude Code: 개발자가 자연어로 코드베이스를 조작할 수 있게 했습니다

각 단계는 "아이디어에서 실행"까지의 마찰을 줄입니다. Claude Code는 현재 이 경로에서 가장 멀리 나아간 단계입니다.
